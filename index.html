Gerador Sintético de carga de trabalho para ambientes 5G

O que é?

EROS-5 é uma ferramenta de geração de carga de trabalho sintético para redes de computadores. O objetivo principal da ferramenta é prover cargas representativas de aplicações reais e facilitar o processo de geração através do uso de variáveis aleatórias que descrevem o comportamento das aplicações.

Como instalar?

Este projeto foi criado utilizando Sistema Operacional Ubuntu 18.04. Ele não foi testado em outros SO's Para instalar, o primeiro passo é clonar o repositório 

 git clone https://github.com/notopoloko/Eros.git
Entre na pasta, crie um ambiente virtual e ative 
 python3 -m venv /path/to/new/virtual/environment 
 source /path/to/new/virtual/environment/bin/activate
Ou instale diretamente na máquina. Em seguida, instale as dependências necessárias 
 pip3 install -r requirements.txt
E rode o programa 
 python3 main.py

Como utilizar

O programa apresenta 4 aplicações sendo elas, VoIP, Streaming de vídeo, WEB e IoT. Cada uma foi modelada utilizando trabalhos conhecidos na área acadêmica se baseando em séries temporais como mostrado na imagem abaixo.

1. VoIP

Na aba VoIP há 3 parâmetros que devem ser passados:

CODEC: CODEC a ser utilizado na geração de tráfego. As opções disponíveis são:

  • G.729 (CBR: 8kbps)
  • G.729 (CBR: 6.4kbps)
  • G.711 A-Law (CBR)
  • SILK (VBR)
  • iSAC (VBR)

Duração: duração em segundos que cada carga terá.

Número de Cargas: número de cargas a serem geradas.

2. Streaming de vídeo

Nessa aba há 3 parâmetros a serem passados

Codificação de vídeo: codificação média (kbps) de vídeo a ser utilizada.

Tempo de vídeo: duração em segundos de cada carga.

Número de cargas: Número de cargas a serem geradas.

3. WEB

Aqui só são necessários 2 parâmtros:

Tempo da carga: duração em segundos de cada carga.

Numero de cargas a serem geradas: número de cargas a serem geradas.

4. IOT

A geração de tráfego IOT precisa de 5 parâmtros:

Número de dispositivos: número de dispositivos que estão transmitindo mensagens.

Tamanho da mensagem: tamanho das mensagens em bytes (M).

Período individual de transmissão: período no qual um único dispositivo vai transmitir uma mensagem (T). A cada T segundos, uma mensagem de tamanho (M) será transmitida por um único dispositivo.

Tempo total: tempo total da carga.

Número de cargas: número de cargas a serem geradas.

5. Total

Tempo inicial: tempo inicial do agregado das cargas geradas a serem analisadas.

Tempo final: tempo final do agregado das cargas geradas a serem analisadas

Saída do gerador

A saída do gerador está no formato json e se encontra na pasta 'Charges'. Após o processo de geração, cada carga estará com o nome <nome_aplicacao>_charge<numero_carga>.json. Cada json possui campos que descrevem a carga segundo a modelagem apropriada:

init_time: Instante de início da carga em segundos (float).

server_port: Porta de destino das mensagens.

packet_size: Tamanho das mensagens a serem transmitidas. Podem ser de tamanho fixo ou variável (int | list).

time_between_packets: Tempo entre duas transmissões consecutivas em segundos (list<float>)
